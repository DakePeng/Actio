# Actio Frontend UX Design

Date: 2026-04-04
Status: DRAFT

## Overview

Actio is a Tauri desktop app that surfaces voice-detected reminders as a minimal sticky-note board. The Rust backend handles ASR and priority ranking. The frontend stays quiet — it speaks only when it has something to say.

## Surfaces

Four surfaces. Nothing more.

1. **Main Board** — Full grid of reminder cards, ranked by priority
2. **System Notifications** — Native desktop toasts (macOS/Windows)
3. **Standby Tray Widget** — Always-visible corner widget
4. **Detail Drawer** — Expanded card view

## Interaction Style

Minimal everywhere except one signature moment. No hover bounce, no breathing cards, no ambient motion. The card appearing from voice IS the motion budget.

### The Wow Moment: Card Appears From Voice

When the ASR backend detects a new reminder and pushes it via Tauri event:

1. Tray widget dot flickers briefly
2. On the board, a card materializes from the bottom-right corner
3. Paper-fold animation: card starts flat/transparent, unfolds upward (rotateX from -15deg to 0)
4. Brief indigo border glow fades over 2 seconds — signals "new"
5. Other cards shift via smooth CSS grid reflow (~300ms)
6. Total animation: ~400ms

If the board is hidden: system notification fires instead. Clicking it opens the board with the new card already in its glow phase.

### Swipe to Dismiss

- **Click + drag** — Grab card edge, drag left or right. Card tilts slightly (rotateZ), Done/X icon fades in
- **Trackpad swipe** — Two-finger horizontal swipe over a card
- **Threshold** — ~60px drag distance triggers dismiss
- **Animation** — Card flips away in swipe direction, collapses to a thin line, disappears (~300ms). Grid gaps close smoothly.
- Fires `done` event to Rust backend.

### Card Detail

Click any card to expand in place. Shows:
- Full voice transcript
- Context (when it was detected, from which conversation)
- Label editor (add/remove labels inline)
- Priority override
- Dismiss button

### Keyboard Shortcuts

5 shortcuts. Only fire when focus is not inside an input field.

| Shortcut | Action |
|----------|--------|
| `Cmd+K` | Focus search |
| `Cmd+N` | New reminder (inline input) |
| `Esc` | Close anything open |
| Arrow keys | Navigate between cards |
| `Enter` | Open/close card detail |

### New Reminder Input

Triggered by `Cmd+N` or the '+' FAB. A quick input slides up from the bottom of the board. User types the reminder text, can assign a label and priority, then submits. Inline, no modal, no page change.

## Standby Tray Widget

Always visible in the bottom-right corner of the screen.

### States

1. **Collapsed** — Logo, pulsing dot, badge count. ~50px wide
2. **Hover-expanded** — Shows top 3 priority items as compact rows. Each row has a priority dot, title, and due time
3. **Click-expanded** — Full list of all items + "View full board" button. Clicking any row opens the board

### New Item Behavior

- Badge count increments
- Dot pulses faster for 3 seconds
- Small number animation on badge

### Transitions

No bounce, no pop. Widget grows in place like a drawer opening. Smooth width/height transitions.

## Card Design

### Priority Badges

- High — Red background (#FEF2F2), red text (#DC2626)
- Medium — Amber background (#FFFBEB), amber text (#D97706)
- Low — Green background (#F0FDF4), green text (#16A34A)

### Labels

Colored chips on each card. Click to filter the board by that label.

| Label | Background | Text |
|-------|-----------|------|
| Work | #EEF2FF | #6366F1 |
| Urgent | #FEF2F2 | #DC2626 |
| Meeting | #FFFBEB | #D97706 |
| Personal | #F0FDF4 | #16A34A |
| Health | #FFFBEB | #CA8A04 |
| Finance | #F0F9FF | #0284C7 |

Custom labels: user-defined name + color picker.

### Card Fields

- Priority badge
- Title (15px, semibold)
- Description (13px, secondary color)
- Due time (11px, tertiary color)
- Labels (colored chips)
- Expandable detail section (voice transcript, context)

## Empty State

**Normal empty:** Centered text: "All clear. I'm listening." with the pulsing dot. No cards, no tips, no illustrations.

**First launch:** Same empty state plus one translucent card: "Start talking. I'll catch what matters." Fades out after 5 seconds or on first voice input. No multi-step onboarding.

## Settings

Accessible via right-click on tray icon or `Cmd+,`.

- Microphone source selector
- Notification preferences (on/off)
- Theme toggle (light/dark)

## System Notifications

Native desktop toasts via `tauri-plugin-notification`.

### When They Fire

- New high-priority item detected from voice
- Reminder approaching (30 min before due time)
- End-of-day summary (if items remain incomplete)

### Notification Content

- App icon + "Actio" label
- Item title
- One-line body with context
- Two actions: "View on Board" / "Dismiss" (or "Snooze" for time-based)

### Click Behavior

Clicking a notification opens the board with the relevant card highlighted.

## Tech Stack

- **Runtime:** Tauri v2 (Rust backend + webview frontend)
- **Frontend:** React + Vite + TypeScript
- **State:** Zustand or React Context
- **Styling:** Tailwind CSS
- **Animations:** CSS transitions + Framer Motion (only for the card-appear and swipe-dismiss)
- **Desktop features:** tauri-plugin-notification, system tray API
- **Backend communication:** Tauri invoke (commands) + Tauri events (push from Rust)
